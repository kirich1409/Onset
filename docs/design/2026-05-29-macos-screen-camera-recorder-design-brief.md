# Design Brief для Claude Design — Onset (нативный macOS-рекордер экрана + камеры)

> Название продукта: **Onset** (от «on set» — бренд нейтрален к функциям, масштабируется на будущие режимы: композиция/наложение).
> Источник истины: `docs/specs/2026-05-29-macos-screen-camera-recorder.md` (MVP v1, status: approved).
> Цель брифа: сгенерировать визуальный дизайн приложения Onset (экраны, состояния, компоненты, интеракции, логотип/иконка) для нативного macOS-приложения.

## 1. Что это за продукт (одним абзацем)

Утилита для macOS (Apple Silicon, macOS 26 «Tahoe»+), которая одновременно записывает экран, внешнюю камеру и микрофон в **раздельные синхронизированные файлы** для последующего монтажа. Пользователь — видеоавтор/стример/создатель туториалов. Сценарий: запустил → выбрал источники и папку → нажал Record → окно свернулось → управляет из menu bar → Stop → готовые файлы открылись в Finder. Тон: профессиональный, утилитарный, быстрый; не «игрушка», но и не перегруженный пультом OBS. Ближайшие референсы по духу: CleanShot X (чистота), Screen Studio (современность), нативные Apple-утилиты (Screenshot.app).

## 2. Платформа и визуальный язык

- **Нативный macOS 26 «Tahoe»** design language: системные материалы/translucency (Liquid Glass), скругления, vibrancy. Только системные шрифты (SF Pro / SF Pro Rounded для акцентов), SF Symbols для иконок.
- **Light и Dark** темы обязательны (равноценные макеты для обеих).
- Стандартные AppKit/SwiftUI контролы (NSPopUpButton/Picker, Toggle, кнопки) — не изобретать кастомные там, где есть системный аналог. Кастомное — только превью камеры и индикаторы записи.
- Размер окна настроек: компактное, фиксированной ширины ~ 480–560 pt, высота по контенту. Не полноэкранное.
- Accessibility: полная клавиатурная навигация, VoiceOver-метки, контраст по WCAG AA, Dynamic Type где применимо.

## 3. Экраны и поверхности (полный список — нужны макеты для каждого)

### 3.1 Главное окно настроек (Settings / pre-record) — ключевой экран
Состоит из секций сверху вниз:

1. **Источник: Экран**
   - Toggle «Запись экрана» (вкл/выкл).
   - Picker дисплея (список подключённых; для каждого — имя + разрешение, напр. «Внешний дисплей — 3840×2160 @ 60 Hz»).
   - (в v1 только весь дисплей; место под будущие режимы «область/окно» предусмотреть, но в MVP не показывать или показать disabled с пометкой «скоро».)

2. **Источник: Камера**
   - Picker камеры: список устройств + первый пункт «Без камеры».
   - Pickers «Разрешение» и «Частота кадров» — заполняются ТОЛЬКО реально поддерживаемыми камерой комбинациями (напр. для MX Brio: 4K · 30 fps / 1080p · 60 fps / 720p · 90 fps). Недоступные комбинации — отсутствуют в списке.
   - **Live-превью камеры** — прямоугольная область 16:9 под пикерами; при «Без камеры» — скрыта или плейсхолдер.

3. **Источник: Микрофон**
   - Picker микрофона: список + «Без звука».
   - (опц.) маленький индикатор уровня входного сигнала (VU-метр) — nice-to-have, показать в макете как вариант.

4. **Вывод**
   - Picker кодека: HEVC (по умолчанию) / H.264.
   - Picker контейнера: MOV (по умолчанию) / MP4.
   - Недоступные на железе комбинации — **показаны задизейбленными с поясняющей подсказкой** (tooltip/инлайн-текст), не скрыты. Пример состояния: опция серая + иконка info + текст причины.
   - Выбор папки сохранения: путь + кнопка «Выбрать…» (стандартный NSOpenPanel).
   - (опц., в раскрывающемся «Дополнительно») override битрейта.

5. **Подвал**
   - Главная кнопка **Record** (акцентная, крупная). Активна ТОЛЬКО при валидной конфигурации (выбран ≥1 видеоисточник). В неактивном состоянии — подсказка, почему (напр. «Выберите экран или камеру»).
   - Слева/рядом — компактный статус валидации/предупреждений (напр. «5K ProRes недоступно на этом чипе» — но ProRes вне scope; реальный пример: «Конфигурация будет авто-скорректирована: …»).

Нужны макеты: дефолт (всё выбрано, Record активна), и компактные варианты секций.

### 3.2 Menu bar (NSStatusItem) — во время записи
- Иконка в строке меню: индикатор «идёт запись» (напр. красная точка/символ записи). При ненулевых дропах — заметный признак деградации (смена цвета/иконки).
- Dropdown по клику: прошедшее время записи (таймер MM:SS), счётчик дропнутых кадров с причиной (когда >0), пункт **Stop** (с показанным key-equivalent глобального hotkey, напр. ⌘⌥⇧R).
Нужны макеты: иконка в баре (нормальная запись / деградация), раскрытый dropdown.

### 3.3 Окно во время записи (восстановленное из Dock)
- Главное окно при Record сворачивается (minimize). Если пользователь кликнул Dock-иконку — окно возвращается в «recording»-виде: крупный таймер, кнопка **Stop**, счётчик дропов; все конфигурационные контролы **задизейблены** с пометкой «недоступно во время записи».
Нужен макет этого состояния.

### 3.4 Системные уведомления (UserNotifications)
- Старт записи (опц.), Stop/готово («Запись сохранена → <папка>»), ошибка источника/writer'а («Камера отключена — файл сохранён частично», «Диск переполнен — поток экрана остановлен»).
Нужны макеты текстов/иконок уведомлений (можно как список вариантов).

### 3.5 Состояния разрешений (TCC)
- Первый запуск: запросы Screen Recording, Camera, Microphone, Notifications.
- **Denied-состояния** в окне настроек: если разрешение не выдано — соответствующий источник недоступен, с понятной подсказкой и кнопкой «Открыть Системные настройки».
Нужны макеты: инлайн-баннер «Нет доступа к экрану/камере/микрофону» с CTA.

### 3.6 Empty / edge states
- Нет подключённых камер → picker показывает только «Без камеры», превью-плейсхолдер.
- Нет микрофонов → «Без звука».
- Папка вывода не выбрана / недоступна для записи → предупреждение.
- Авто-коррекция конфигурации (AC-15) → ненавязчивое уведомление «Параметры скорректированы под возможности устройства: <что именно>».
Нужны макеты для каждого.

## 4. Инвентарь компонентов (для дизайн-системы)
- Секция-карточка источника (заголовок + контролы), с состоянием вкл/выкл.
- Picker устройства (имя + метаданные строкой).
- Превью-вьюпорт камеры (16:9, с плейсхолдером и рамкой).
- Disabled-опция с info-иконкой и tooltip-причиной.
- Кнопка пути (truncate середины пути).
- Главная акцентная кнопка Record / Stop.
- Индикатор записи (menu bar icon: idle/recording/degraded).
- Таймер записи (моноширинные цифры).
- Счётчик дропов с причиной (бейдж/чип).
- Инлайн-баннер разрешений/предупреждений с CTA.
- VU-метр уровня микрофона (опц.).

## 5. Ключевые интеракции (для прототипа/спецификации поведения)
- Смена камеры → превью мгновенно переключается; пикеры разрешения/fps пересобираются под новую камеру.
- Выбор невозможной комбинации кодек/разрешение → опция disabled с причиной (не молчаливое исчезновение там, где это «не тянет железо»; для камеры — реально неподдерживаемое просто отсутствует).
- Record → окно minimize, появляется/активируется menu bar индикатор.
- Stop доступен 3 способами: menu bar, глобальный hotkey, Dock-иконка (возврат окна). Все три отразить в дизайне.
- Hotkey показывается как key-equivalent рядом со Stop.
- Hotplug устройства (подключение/отключение) во время настроек → списки обновляются.

## 6. Чего НЕ делать (out of scope для дизайна v1)
- Реал-тайм композиция/PiP (камера поверх экрана) — раздельные файлы, превью НЕ накладывается на экран.
- Режимы «область экрана» и «отдельное окно» — Phase 2 (можно зарезервировать место, но не проектировать UI выбора области/окна сейчас).
- Системный звук, ProRes, HDR, редактор/таймлайн, шеринг — вне продукта/вне v1.
- Countdown, аннотации, эффекты курсора — вне v1.

## 7. Что нужно на выходе от Claude Design
1. Макеты всех экранов/состояний из §3 в Light и Dark.
2. Состояния главной кнопки (Record активна/неактивна, Stop).
3. Menu bar icon в трёх состояниях (idle/recording/degraded) + dropdown.
4. Дизайн-токены (цвета, типографика, отступы, радиусы) в духе macOS 26.
5. Инвентарь компонентов (§4) как переиспользуемые элементы.
6. Спецификация поведения интеракций (§5) — аннотации на макетах.
7. **App icon / логотип Onset** — иконка приложения (macOS rounded-rect app icon, набор размеров) и menu bar template-иконка (монохром, под `NSStatusItem`). Идея: «on set» / запись / съёмочный кадр; работает в Light/Dark и как 16pt template в строке меню.

## 8. Тон и эстетика (резюме)
Минималистичный, профессиональный нативный macOS-инструмент. Приоритет — скорость до записи (минимум кликов от запуска до Record) и спокойная, не отвлекающая поверхность во время записи (вся индикация уходит в menu bar). Никакого «стримерского пульта». Чистые секции, ясная иерархия, акцент на кнопке Record и на превью камеры.
