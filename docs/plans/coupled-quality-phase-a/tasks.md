- acceptance: THE SYSTEM SHALL map a CPU brand string to a `ChipTier` purely and totally: a string identifying Apple M3 Max (e.g. "Apple M3 Max", case/space-normalized) ‚Üí `.m3Max`; **every other input** ‚Äî other Apple chips ("Apple M3", "Apple M3 Pro", "Apple M2 Max", ‚Ä¶), empty, garbage, non-Apple ‚Üí `.uncalibrated`. It SHALL NEVER crash on any input and SHALL classify anything not-provably-M3-Max as `.uncalibrated` (never up). `detectChipTier()` reads `machdep.cpu.brand_string` via `sysctlbyname` and delegates to the pure function. `hw.model` is NOT used. **Concrete recipe (empirically confirmed: exit 0, 0 warnings under the project flags; `import Foundation` suffices, no `import Darwin`):**
  ```swift
  var size = 0
  let r1 = unsafe sysctlbyname("machdep.cpu.brand_string", nil, &size, nil, 0)
  guard r1 == 0, size > 0 else { return .uncalibrated }
  var buffer = [CChar](repeating: 0, count: size)
  let r2 = unsafe buffer.withUnsafeMutableBufferPointer { unsafe sysctlbyname("machdep.cpu.brand_string", $0.baseAddress, &size, nil, 0) }
  guard r2 == 0 else { return .uncalibrated }
  let trimmed = buffer.prefix { $0 != 0 }.map { UInt8(bitPattern: $0) }   // drop the NUL sysctl includes in `size`
  let raw = String(decoding: trimmed, as: UTF8.self)                       // NOT String(cString:) ‚Äî deprecated ‚Üí warnings-as-errors build FAIL
  ```
  Three separate `unsafe` sites are required (both `sysctlbyname` calls + the buffer-pointer closure); the NUL must be trimmed or the parse-matrix comparison silently fails. Log `raw` via the `ChipTierDetector` `Logger`.
