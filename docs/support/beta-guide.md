# Onset Beta — Инструкция для первых пользователей

> Документ актуален для сборки 0.1.0 (macOS 26+).

---

## 1. Первый запуск и разрешения

Onset требует три разрешения TCC. macOS запрашивает их при первом обращении к каждому ресурсу.
Если разрешение было отозвано и выдано повторно — приложению может потребоваться перезапуск,
чтобы увидеть изменение (поведение платформы macOS, не баг приложения).

| Разрешение | Где выдать | Когда нужно |
|---|---|---|
| **Запись экрана** | System Settings → Privacy & Security → Screen Recording → Onset | при старте первой записи |
| **Камера** | System Settings → Privacy & Security → Camera → Onset | при старте первой записи с камерой |
| **Микрофон** | System Settings → Privacy & Security → Microphone → Onset | при старте первой записи с микрофоном |

После выдачи разрешения **«Запись экрана»** приложение автоматически перезапускается —
это штатное поведение.

---

## 2. Как записывать

1. Откройте главное окно Onset (двойной клик на иконке в строке меню, или ⌘⌥⌃R для переключения).
2. Выберите источники:
   - **Дисплей** — дисплей, который будет записываться.
   - **Камера** — встроенная или внешняя камера (можно отключить).
   - **Микрофон** — аудиоисточник (можно отключить).
3. Нажмите **«Записать»**.

Если выбранная камера была отключена (например, вынута из USB-порта), под строкой выбора устройства появляется уведомление «Камера "X" недоступна…». При наличии других камер пикер остаётся активным — выберите другое устройство или подключите нужное обратно.

### Остановка записи

Остановить запись можно тремя способами:

- Кнопка **«Стоп»** в главном окне.
- Глобальный хоткей **⌘⌥⌃R** (работает из любого приложения).
- Иконка Onset в **строке меню** → «Остановить запись».

## 3. Результат записи

Каждая запись сохраняется в **отдельную подпапку** внутри базовой папки:

```
~/Movies/Onset/
└── Onset 2026-06-12 14.30.05/
    ├── Onset 2026-06-12 14.30.05 — Screen.mp4
    └── Onset 2026-06-12 14.30.05 — Camera.mp4
```

По умолчанию базовая папка — `~/Movies/Onset/`. Изменить её можно в главном окне Onset,
секция **«ВЫВОД»** → строка «Папка» → кнопка **«Выбрать…»**. Выбор сохраняется между запусками.

| Файл | Содержимое |
|---|---|
| `Onset … — Screen.mp4` | Запись экрана, HEVC |
| `Onset … — Camera.mp4` | Запись камеры, HEVC |

Оба файла получают дорожку микрофона (один микрофон на оба файла — MVP).
Файлы совместимы с QuickTime Player, iMovie и Final Cut Pro.

После остановки записи Finder автоматически открывает папку сессии.

## 4. Рекомендуемая конфигурация для беты

Тестирование и разработка ведутся на следующем стенде:

- **Дисплей:** реальный внешний дисплей или встроенный экран MacBook; Retina-дисплеи (5K/6K Pro Display XDR) могут показывать повышенный счётчик дропов — см. [Known Issues](known-issues.md).
- **Камера:** встроенная FaceTime HD (1080p30) **или** Logitech MX Brio (1080p, ~20 fps — см. [Known Issues](known-issues.md)). Другие USB-камеры должны работать, но не тестировались систематически.
- **Формат:** 1080p 16:9 — единственный поддерживаемый MVP-формат; пикер режимов отсутствует.
- **macOS:** 26.x (Sequoia) на Apple Silicon.

## 5. Диагностика и обратная связь

### Журнал событий

Если что-то пошло не так, экспортируйте журнал до перезапуска приложения:

1. Нажмите на значок Onset в строке меню.
2. Выберите **«Экспортировать диагностику»**.
3. Выберите папку — файл `onset-diagnostics-ГГГГ-ММ-ДД-ЧЧММСС.log` откроется в Finder.

Подробнее: [docs/support/diagnostics.md](diagnostics.md)

### Crash-логи (`.ips`)

При аварийном завершении macOS сохраняет отчёт в `~/Library/Logs/DiagnosticReports/Onset-*.ips`.

1. В Finder нажмите **Перейти → Перейти к папке…** (`⇧⌘G`).
2. Введите `~/Library/Logs/DiagnosticReports/` → нажмите **Перейти**.
3. Найдите файл `Onset-*.ips` по дате и приложите к описанию проблемы.

### Версия приложения

Версия видна в меню Onset (строка меню → Onset). Обязательно указывайте её при отправке обратной связи.
